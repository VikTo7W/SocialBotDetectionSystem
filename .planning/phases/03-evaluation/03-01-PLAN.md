       - F1: f1_score(y_true, (p_final >= threshold).astype(int))

       - For each of ["p1", "p2", "p12", "p_final"]:
         - Same four metrics (F1, AUC, precision, recall) computed against y_true

    4. Compute routing statistics (EVAL-03):
       - Stage 1 exit: accounts where amr_used==0 AND stage3_used==0 (exited after Stage 1+2 meta without needing AMR or Stage 3)

         - ALL accounts get meta12 combination
         - Some get routed to Stage 3 (stage3_used==1)
         - So "exit stages" = percentage that did NOT use stage3 (exited at meta12 level), percentage that DID use stage3
         - Report: pct_amr_triggered = 100 * sum(amr_used) / n
         - pct_stage3_used = 100 * sum(stage3_used) / n
         - pct_early_exit = 100 * sum((stage3_used==0)) / n (exited without Stage 3)
       - Store as: pct_stage1_exit (didn't need AMR or Stage 3 — amr_used==0 AND stage3_used==0), pct_stage2_exit (used AMR but not Stage 3 — amr_used==1 AND stage3_used==0), pct_stage3_exit (used Stage 3 — stage3_used==1), pct_amr_triggered (any AMR use)
