# Plan 19-02

## Objective

Build the maintained BotSim training entry point and demote `main.py` from maintained training surface to compatibility or legacy-wrapper status.

## Tasks

1. Extract any reusable BotSim data-loading or split helpers needed from `main.py` without reintroducing duplicated training orchestration.
2. Implement `train_botsim.py` as the maintained BotSim training command using `CascadePipeline("botsim")`.
3. Change the default output artifact to `trained_system_botsim.joblib` and ensure legacy artifacts are not written by default.
4. Keep `main.py` only as a thin compatibility path if still needed for existing manual workflows.

## Verification

- `python -m py_compile train_botsim.py main.py tests/test_train_botsim.py`
- targeted pytest for BotSim training entry-point tests

## Acceptance Criteria

- Running `train_botsim.py` is the maintained path for BotSim training.
- The default artifact is `trained_system_botsim.joblib` and TwiBot artifacts are never touched.
- The BotSim path remains a thin script over shared pipeline logic rather than a second orchestration implementation.
