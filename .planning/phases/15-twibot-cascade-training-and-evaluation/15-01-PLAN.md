<objective>
Implement the TwiBot-native cascade training entry point so v1.4 can produce a separate
Twitter-trained `TrainedSystem` artifact without disturbing the existing Reddit-trained model path.

Purpose: satisfy `TRN-01` and `TRN-03` by wiring the Phase 14 native features into the
existing cascade training stack with explicit split discipline and separate artifact naming.

Output:
- `train_twibot20.py`
- Focused training-path tests
</objective>

<tasks>

<task>
  <name>Task 1: Build the TwiBot-native training/calibration entry point</name>
  <files>train_twibot20.py</files>
  <action>
    Create a dedicated TwiBot-native training entry point. It should:
    - load `train.json`, `dev.json`, and `test.json` through `twibot20_io`
    - split `train.json` into S1 and S2 with label stratification and fixed seed
    - filter intra-split edges locally
    - scope in the Phase 14 native extractors for Stage 1 and Stage 2 using a safe override/restoration pattern
    - call `train_system()`
    - calibrate thresholds on `dev.json`
    - save a separate TwiBot-native joblib artifact and any directly related training-side metrics/evidence artifacts

    Do not overwrite `trained_system.joblib`, `trained_system_v11.joblib`, or `trained_system_v12.joblib`.
    Use the repo-visible split names (`train.json`, `dev.json`, `test.json`) rather than `val.json`.
  </action>
  <verify>
    Prove the script preserves split discipline, restores overridden extractors in `finally`, and saves to a distinct artifact path.
  </verify>
</task>

<task>
  <name>Task 2: Add focused training-path tests</name>
  <files>tests/test_train_twibot20.py</files>
  <action>

    - S1/S2 split construction and deterministic seed behavior
    - edge filtering behavior for split-local graphs
    - scoped native extractor override and restoration
    - separate artifact path selection that does not collide with the Reddit-trained joblib names
    - clear handling of the repo-local `dev.json` calibration split contract
  </action>
  <verify>

  </verify>
</task>

</tasks>
