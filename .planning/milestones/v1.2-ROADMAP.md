# Milestone v1.2: TwiBot-20 Cross-Domain Transfer

**Status:** SHIPPED 2026-04-18
**Phases:** 8-10
**Total Plans:** 5

## Overview

v1.2 extended the Reddit-trained cascade to a zero-shot TwiBot-20 transfer path without retraining. The milestone replaced brittle demographic-proxy assumptions with behaviorally grounded Twitter mappings, added missingness-aware handling for systematically absent TwiBot fields, introduced online novelty-threshold recalibration, and produced comparison/table generation paths for the paper.

## Phases

### Phase 8: Behavioral Tweet Parser and Transfer Adapter Stabilization

**Goal**: The system can accept a TwiBot-20 account's tweet/domain data and produce transfer-stable features for zero-shot inference without retraining
**Depends on**: Phase 7
**Plans**: 2 plans

Plans:

- [x] 08-01-PLAN.md - Add `parse_tweet_types()` in `twibot20_io.py` with RT/MT/original parsing coverage
- [x] 08-02-PLAN.md - Rewrite the transfer adapter in `evaluate_twibot20.py`, add transfer diagnostics, review `_RATIO_CAP`, and verify TwiBot behavior

**Details:**

- `parse_tweet_types()` classifies RT, MT, and original tweets and extracts usernames for RT/MT handling
- TwiBot loader exposes `domain_list` and Twitter graph relations for transfer inference
- Stage 1 mapping was revised to use tweet behavior plus `domain` rather than the older demographic-proxy path
- Stage 2 timestamp-missing features now use missingness-aware sentinel handling rather than ordinary zeros

### Phase 9: Sliding-Window Online Threshold Recalibration

**Goal**: The inference pipeline adjusts routing thresholds unsupervised as accounts are processed, using only novelty scores from a running buffer
**Depends on**: Phase 8
**Plans**: 1 plan

Plans:

- [x] 09-01-PLAN.md - Add chunked online calibration to `run_inference()` with configurable window size and threshold restoration

**Details:**

- `run_inference()` now supports `online_calibration` and `window_size`
- novelty thresholds `n1_max_for_exit`, `n2_trigger`, and `novelty_force_stage3` are updated from buffered novelty scores only
- the trained system's original threshold object is restored before return

### Phase 10: Cross-Domain Evaluation and Paper Output

**Goal**: Quantitative evidence of zero-shot transfer performance on TwiBot-20 is produced and formatted for the paper's robustness section
**Depends on**: Phases 8 and 9
**Plans**: 2 plans

Plans:

- [x] 10-01-PLAN.md - Add static-vs-recalibrated TwiBot comparison and persist comparison artifacts
- [x] 10-02-PLAN.md - Generate the final cross-dataset LaTeX table from BotSim-24 plus both TwiBot conditions

**Details:**

- `compare_twibot20_conditions()` compares static thresholds against online recalibration on the revised adapter
- `metrics_twibot20_comparison.json` is now the stable comparison artifact for downstream reporting
- `ablation_tables.py` can generate Table 5 with BotSim-24, TwiBot static, and TwiBot recalibrated columns

## Milestone Summary

**Key Decisions:**

- Reopened the frozen Phase 8 Twitter mapping after the saved TwiBot run collapsed to near-all-human predictions
- Allowed TwiBot `domain` to serve as the best available `subreddit_list` analog for topical breadth
- Preserved feature dimensionality while using missingness-aware sentinels for systematically absent TwiBot timestamps
- Kept v1.2 zero-shot only: no TwiBot retraining and no full Twitter-native redesign

**Issues Resolved:**

- Removed the earlier demographic-proxy transfer assumptions from the active TwiBot path
- Added a stable before/after comparison path for static vs recalibrated TwiBot transfer
- Added a paper-ready cross-dataset table builder that reflects the live milestone semantics

**Issues Deferred:**

- Fresh real-data TwiBot comparison artifacts still need to be generated
- Full milestone audit was not run before close
- Final pytest/runtime verification remains partially blocked by local Windows temp and process-permission issues

**Technical Debt Incurred:**

- Phase 10 comparison code is in place, but the evidence artifact still depends on a successful fresh run in a cleaner environment
- The requirements file was not maintained as a live checked-off source during execution and had to be archived retroactively
