# Requirements - v1.2 TwiBot-20 Cross-Domain Transfer

## Requirement Outcomes

### FEAT - Behavioral Feature Adapter

- [x] **FEAT-01**: Behavioral tweet parser classifies each account's tweets into RT-prefixed, MT-prefixed, and regular (original) buckets, and extracts distinct @usernames from RT/MT tweets
  Outcome: Implemented via `parse_tweet_types()` and covered by TwiBot parser tests.
- [x] **FEAT-02**: Stage 1 transfer adapter uses a documented TwiBot-to-BotSim mapping that materially improves zero-shot transfer over the demographic proxy baseline; the chosen mapping may revise how `submission_num`, `comment_num_1`, `comment_num_2`, and `subreddit_list` are sourced
  Outcome: Mapping was revised to total/original/MT/domain semantics. Qualitative improvement work is implemented, but fresh real-data evidence is still pending.
- [x] **FEAT-03**: Ratio cap is reviewed and tuned for Twitter tweet-count distributions (current `1000.0` may need adjustment with the new behavioral counts)
  Outcome: Reviewed and retained at `1000.0` with documented rationale and transfer diagnostics.
- [x] **FEAT-04**: When TwiBot fields are systematically unavailable but not semantically zero, Phase 8 may apply dimensionality-preserving missingness-aware handling for transfer stability without retraining
  Outcome: Implemented using sentinel handling for timestamp-derived Stage 2 features.

### CAL - Online Threshold Calibration

- [x] **CAL-01**: Sliding-window threshold recalibrator updates all routing thresholds every N accounts using a running buffer of per-account novelty scores (unsupervised - labels never used during inference)
  Outcome: Implemented for novelty routing thresholds only (`n1_max_for_exit`, `n2_trigger`, `novelty_force_stage3`), matching the approved v1.2 scope.
- [x] **CAL-02**: Window size N is configurable (default suggested: 100 accounts)
  Outcome: Implemented with `window_size=100` default.
- [x] **CAL-03**: Calibrator degrades gracefully when the buffer has fewer than N accounts (cold-start: use original trained thresholds until buffer fills)
  Outcome: Implemented and covered by cold-start tests.

### EVAL - Evaluation and Paper Output

- [x] **EVAL-01**: Before/after comparison reports F1, AUC, precision/recall on TwiBot-20 with (a) current demographic proxy adapter and (b) revised transfer adapter
  Outcome: Adjusted at execution time to the active milestone semantics: revised adapter with static thresholds vs revised adapter with online recalibration. Comparison code and artifacts are implemented, but a fresh real-data run is still pending.
- [x] **EVAL-02**: Paper-ready LaTeX cross-dataset results table comparing BotSim-24 in-domain performance vs TwiBot-20 zero-shot transfer performance
  Outcome: Implemented in `ablation_tables.py` using BotSim-24 plus TwiBot static/recalibrated columns.

| Profile/description features | Permanently excluded after the leakage audit |
