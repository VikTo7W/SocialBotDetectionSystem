## v1.3 Requirements

### Evaluation Reproducibility

- [x] **REPRO-01**: Developer can run a documented TwiBot evaluation command that produces both static-threshold and recalibrated outputs in a reproducible local artifact directory
  - **Outcome:** Delivered in Phase 11. `evaluate_twibot20.py` now accepts `output_dir` argument with `os.makedirs` routing. Canonical command documented in module docstring. Default output_dir is the Phase 12 milestone artifact directory.

- [x] **REPRO-02**: TwiBot evaluation no longer depends on fragile default temp/cache paths for its normal artifact-generation workflow
  - **Outcome:** Delivered in Phase 11. All artifact writes route through `os.path.join(output_dir, ...)`. Production code confirmed to contain zero `tempfile`/`gettempdir` usage.

- [x] **REPRO-03**: Generated artifacts include results, metrics, and comparison outputs with stable filenames and documented meaning
  - **Outcome:** Delivered in Phase 11-12. `EXPECTED_OUTPUT_FILES` constant added; all four artifact filenames and payload schemas documented in module docstring.

### Transfer Evidence

- [x] **EVID-01**: A fresh TwiBot run generates live static-vs-recalibrated metrics from the current revised adapter
  - **Outcome:** Delivered in Phase 12. Live run produced: static F1=0.0/AUC=0.5964, recalibrated F1=0.0/AUC=0.5879. Artifacts written to `.planning/workstreams/milestone/phases/12-fresh-transfer-evidence-and-paper-outputs/artifacts/`.

- [x] **EVID-02**: The final cross-dataset LaTeX table is generated from fresh BotSim-24 and TwiBot artifacts rather than stale placeholder outputs
  - **Outcome:** Delivered in Phase 12. `ablation_tables.py` now defaults to Phase 12 artifact path. `tables/table5_cross_dataset.tex` regenerated from live comparison.

- [x] **EVID-03**: The milestone records whether recalibration materially improves TwiBot transfer quality and documents the observed outcome
  - **Outcome:** Delivered in Phase 12. Verdict: `no_material_change` (basis: `f1_delta`). Recalibration increases Stage 3 routing on TwiBot accounts but does not improve final F1.

### System Versioning

- [x] **VERS-01**: The chosen TwiBot system version is explicitly identified, including the model artifact(s), evaluation mode(s), and expected output files
  - **Outcome:** Delivered in Phase 13. `VERSION.md` created at project root naming `trained_system_v12.joblib`, both eval modes (static/recalibrated), all 6 output files, live verdict, and all 3 env-var overrides.

- [x] **VERS-02**: README or milestone-facing docs explain how to reproduce the TwiBot system-version outputs end to end
  - **Outcome:** Delivered in Phase 13. `README.md` expanded with numbered Reproduction Guide containing exact fenced bash commands for the canonical Phase 12 invocation.

- [x] **VERS-03**: Remaining execution caveats, environment assumptions, and known limitations are documented with the released system version
  - **Outcome:** Delivered in Phase 13. `README.md` contains Known Caveats (Windows pytest, TwiBot F1=0.0, AMR stub, calibration artifact) and Known Limitations (6 out-of-scope items).
