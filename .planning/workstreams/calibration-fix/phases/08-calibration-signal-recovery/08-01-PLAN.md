Add structured trial diagnostics inside `calibrate_thresholds()` without breaking its public return type.

1. Compute and store, per trial:
   - primary metric score (existing objective)
   - predicted hard labels from `p_final >= 0.5`
   - count of positive predictions
   - AMR usage rate and Stage 3 usage rate from `predict_system()` outputs
   - at least one smooth secondary metric derived from `p_final` and `y_true` such as log loss or Brier score
2. Attach these diagnostics to the Optuna trial using `trial.set_user_attr(...)` so later selection/reporting logic can inspect them without rerunning inference.
3. Add concise end-of-study reporting that summarizes:
   - best primary score
   - how many trials matched the best primary score
   - whether those best-score trials share identical hard predictions
   - whether routing behavior varies across the tied trials
4. Do not change the current public return type yet; this plan is for diagnosis and instrumentation.
