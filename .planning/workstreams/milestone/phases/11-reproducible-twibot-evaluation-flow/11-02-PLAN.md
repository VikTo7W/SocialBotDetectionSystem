<objective>
Harden the TwiBot evaluation artifact path so static and recalibrated outputs land in
a documented, developer-specified directory (defaulting to cwd), document the canonical
command and artifact payload schemas in the module docstring, and let ablation_tables.py
discover the comparison artifact via an env var so downstream tooling does not depend on
cwd.

Purpose:
  - REPRO-01: Developer can run one documented command that produces both static and
    recalibrated outputs in a reproducible local directory.
  - REPRO-02: Artifact writes do not rely on fragile cwd-relative bare filenames.
  - REPRO-03: The three artifact filenames and payload schemas are documented in the
    module docstring so downstream tooling (ablation_tables.py Table 5, Phase 12) can
    consume them reliably.

Output:
  - evaluate_twibot20.py with an `output_dir` positional argument on __main__ (default
    "."), all three artifact writes routed through `os.path.join(output_dir, ...)`, and
    an expanded module docstring.
  - ablation_tables.py with `TWIBOT_COMPARISON_PATH` env-var override for the
    comparison artifact location.
  - No new tests; no production test regressions.

Note on stale artifacts (RESEARCH Pitfall 5): do NOT delete existing root-level
`results_twibot20.json` or `metrics_twibot20.json`. They are stale pre-Phase-8 artifacts
but Phase 11 is path hardening only; Phase 12 is responsible for fresh runs.
